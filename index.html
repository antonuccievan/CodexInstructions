Overview

From Idea to Live Site with Codex + GitHub Pages

This presentation walks through your exact process for creating a public repository, developing with Codex, merging via PR, and publishing to GitHub Pages.

Step 1

Create a Public GitHub Repository

  1. Go to GitHub and create a new repository.
  2. Set repository visibility to Public.
  3. Add a README if desired, then create the repo.
Step 2

Create a Codex Environment

  1. In Codex, open Settings → Environment.
  2. Click Create Environment.
  3. Select your new GitHub repository.
  4. Set sharing to Only You (important).
Step 3

Build in Codex

  1. Use the connected repository environment in Codex.
  2. Ask Codex to build your project (for example, a game or website).
  3. Iterate until Codex finishes the requested work.
Step 4

Create and Review PR

  1. Click CreatePR in Codex.
  2. Then click ViewPR to open it in GitHub.
  3. Review the code changes and checks.
Step 5

Merge and Publish

  1. Merge the pull request in GitHub.
  2. Open repository Settings → Pages.
  3. Set source branch to main and click Save.
  4. Wait for deployment, then visit your live URL.
